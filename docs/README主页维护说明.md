## README 主页维护说明

本仓库的 `README.md` 用作 GitHub 个人主页展示内容（Profile README）。为了保持风格统一与可维护性，建议按下面约定更新。

### 推荐维护方式

- **先改“信息”，再改“样式”**：优先更新项目/技能/联系方式等真实内容，样式（徽章、卡片）尽量少动。
- **保持模块短小**：每个模块 3-6 行为宜，避免大段文字影响阅读。
- **内容尽量可验证**：项目建议放真实仓库链接；技术栈以“常用且能落地”为主。
- **避免敏感信息**：不要把手机号、住址、公司内部信息写进主页。

### 你通常会改到的地方

- **自我介绍**：姓名/称呼、方向、关注领域
- **在做什么**：最近 1-3 个方向/主题即可
- **精选项目**：2-4 个项目，写清楚“做了什么 + 价值”
- **联系我**：建议留 GitHub；如需邮箱/微信，请自行决定是否公开

### 徽章与统计卡片注意事项

- **访问量徽章**：依赖第三方服务，偶尔不可用属于正常现象
- **统计卡片**：来自 `github-readme-stats`、`github-readme-streak-stats` 等服务，偶尔会因限流/缓存导致加载慢
- **用户名统一**：所有统计/徽章链接里的 `username=` 建议统一为你的 GitHub 用户名

### 结构建议（可对照 `README.md`）

- 顶部横幅/标题（可选）
- 名片区（简介 + 徽章）
- 我能做什么 / 我最近在做
- 技术栈
- 精选项目（建议固定 2-4 个）
- 联系方式
- 统计（可选，别太多）
