### 我写代码的习惯

- **先定规范再写代码**：命名、目录结构、日志/错误码、可观测性优先
- **优先复用**：把通用逻辑收敛成工具/组件，避免散落的重复实现
- **可维护性**：宁可多写一点“正确的结构”，也不堆短平快的临时代码
- **可运行**：尽量提供最小可运行示例与一键启动方式（环境/依赖明确）
